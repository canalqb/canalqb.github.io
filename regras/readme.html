Fundação

O que é o Master Rules?

O master_rules.md é o documento central que define todas as regras, templates e padrões que qualquer LLM deve seguir ao criar código PHP para este sistema. É a fonte única de verdade — nenhuma outra instrução o sobrepõe.

"Este arquivo sobrepõe qualquer outra regra! Varrer recursivamente a pasta /regras/ antes de qualquer ação!"

— master_rules.md
ℹ️Ciclo de Leitura Obrigatório para o LLM

Leitura Inicial → Durante Modificações → Rechecagem Final antes de entregar qualquer código.

Visão Geral

Como o Sistema Funciona

01
LLM lê o master_rules.md
Antes de qualquer tarefa, o assistente deve varrer todas as regras disponíveis em /regras/. Sem pular etapas.
02
Identifica o template correto
Para cada tipo de página ou componente, existe um template específico. O LLM seleciona o mais adequado antes de escrever uma linha de código.
03
Personaliza apenas as variáveis
Apenas os campos marcados com {chaves} são modificados. A estrutura e as regras de segurança permanecem intactas.
04
Valida e entrega
Rechecagem final: segurança CSRF aplicada, acessibilidade WCAG AA garantida, temas claro/escuro funcionando, responsividade verificada.

Guia Prático

Como Fazer Pedidos ao LLM

⚠️Antes de tudo

O LLM precisa ter acesso ao master_rules.md no contexto da conversa ou no sistema de arquivos acessível. Sem isso, as regras não serão aplicadas automaticamente.

📄 Solicitar Página Administrativa

prompt 
"Crie uma página de configuração de layout para o meu sistema de vendas.
Use o template estrutura_php_pagina_admin_config_universal.php
e aplique o tema da minha empresa: cor primária #2563eb, nome 'SistemaVendas'."
💡O que acontece

O LLM carrega estrutura_php_pagina_admin_config_universal.php, substitui as variáveis entre {chaves} e entrega uma página funcional com CSRF, temas e AJAX prontos.

🎨 Solicitar Componente UI

prompt 
"Preciso de uma seção de serviços para meu site de consultoria.
Use o componente columns_with_icons.
Serviços: Auditoria, Consultoria Fiscal, Planejamento Tributário, Compliance."

🔧 Solicitar Elemento Específico

prompt 
"Adicione um combobox dinâmico de seleção de categorias nesta página.
Use estrutura_php_elemento_combobox_dinamico.
Endpoint AJAX: /api/categorias, parâmetro de busca: q"

📊 Solicitar Validação de Qualidade

prompt 
"Valide as regras de qualidade deste arquivo conforme regra_llms_metricas_qualidade_severa.md.
Apresente o relatório comparativo mostrando o % Antes e o % Depois."

Garantias do Sistema

Funcionalidades Sempre Presentes

🛡️
Segurança por Padrão
Token CSRF em todos os formulários. Sanitização e validação de entradas. Verificação de permissão em todas as ações sensíveis.
♿
Acessibilidade WCAG AA
ARIA labels em todos os elementos interativos. Contraste de cores verificado em ambos os temas. HTML semântico obrigatório.
🌓
Temas Claro & Escuro
Alternância dinâmica com persistência via localStorage. Contraste acessível garantido nos dois modos sem exceção.
📱
Mobile-first
Bootstrap 5 com breakpoints responsivos. Layouts adaptados para telas de 320px até 2560px sem conteúdo quebrado.
Para o Usuário

Como Obter Melhores Resultados

💬
Seja específico e contextualizado
Em vez de "crie um site", diga "crie um site de gestão de clínicas veterinárias com módulo de agendamento e prontuário digital". Quanto mais contexto, menos revisões.
📎
Cite o template quando souber
Se você sabe qual template usar, mencione explicitamente. Isso elimina perguntas do LLM e agiliza a entrega.
🎨
Informe a identidade visual
Cor primária em hex, nome do sistema e logotipo (se houver). O LLM substitui esses valores nas variáveis dos templates na primeira geração.
🧪
Teste e itere com contexto
Ao pedir ajustes, descreva o comportamento esperado vs. o observado. Isso ajuda o LLM a entender a modificação sem quebrar outras partes.

Protocolo de Relatório Comparativo

📊Nova Regra de Relatório (Antes vs Depois)

Todo trabalho entregue após validação de qualidade deve incluir a tabela de evolução. Isso garante ao usuário que as falhas detectadas foram corrigidas e o código atingiu 100% de conformidade.

MétricaFluxoMeta
Avaliação InicialCheck em todos os arquivosGerar % Antes
Correção SeveraImplementar melhorias e padrõesCorrigir falhas
